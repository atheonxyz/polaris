# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Polaris, please report it responsibly.

**DO NOT** open a public GitHub issue for security vulnerabilities.

Instead, please send an email to: security@atheon.io (or create a private security advisory on GitHub)

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes

## Security Best Practices

When using Polaris:

1. **Protect your recovery phrase**: Never share it with anyone. Store it offline in a secure location.

2. **Use strong passwords**: Your wallet password should be unique and complex.

3. **Verify transactions**: Always double-check addresses and amounts before confirming.

4. **Keep software updated**: Use the latest version of Polaris for security fixes.

5. **Secure your machine**: Ensure your computer is free from malware.

## Scope

This security policy applies to:
- The Polaris CLI application
- Local wallet storage and encryption
- Network communication with RPC providers

## Out of Scope

- Third-party RPC providers
- RAILGUN protocol vulnerabilities (report to RAILGUN team)
- User operational security mistakes
